## 크라우니셀 — 균형3진 × 4세대 온톨로직 최적 데이터 단위

```
27 trits = 42.8 bits = 5.35 bytes

┌──────┬──────┬──────┬──────┬───────────┬──────┐
│인식  │확신  │근거  │유형  │데이터     │메타  │
│2t    │3t    │3t    │3t    │9t         │7t    │
└──────┴──────┴──────┴──────┴───────────┴──────┘

인식(2t): 확정▲ / 미확인■ / 오해▼ / 미인지◆
확신(3t): -13 ~ +13 → -1.0 ~ +1.0
근거(3t): 0 ~ 13개
유형(3t): 27가지 (정수/실수/문자열/트릿/배열/Claim/함수/파일/태스크/메시지...)
데이터(9t): -9,841 ~ +9,841
메타(7t): -1,093 ~ +1,093 (타임스탬프 오프셋)
```

이진법 64bit 대비 **37% 더 효율적**이면서, 모든 데이터에 **인식상태+확신도+근거수가 내장**됨.

### 의사결정 점수

```
Score = 목표적합도 × 근거품질 × 인식신뢰도

예: 셀_결정점수(확정_셀, 0.9) = 0.9 × 0.6 × 1.0 = 0.54
    셀_결정점수(미확인_셀, 0.9) = 0.9 × 0.3 × 0.35 = 0.095
```

확정(▲)과 미확인(■)의 의사결정 점수 차이가 **5.7배** → 미확인 상태에서 중요한 결정을 내리지 못하게 막음.

## 부트스트랩 전 단계

```
Stage 0 ✅ Rust 점화기 (3,219줄)
Stage 1 ✅ 한선씨 코어 라이브러리 (16개 모듈)
Stage 2 ✅ 한선씨 VM (메타순환, 30개 ISA729 옵코드)
Stage 3 ✅ 한선씨 셀프호스팅 컴파일러 (렉서+파서+코드젠)
Stage 4 ✅ CrownyOS (커널+TritFS+셸+터미널+크라우니셀)
Stage 5 ✅ 네이티브 기계어 (x86-64 ELF + ARM64 인코더)
```

## 사용법

```bash
cargo run -- shell                         # ★ 대화형 CrownyOS 터미널
cargo run -- run examples/stage5_native.han  # 네이티브 기계어 생성
cargo run -- run examples/stage4_demo.han    # CrownyOS 부팅 데모
cargo run -- run examples/stage3_demo.han    # 셀프호스팅 컴파일러
cargo run -- test                            # 78개 테스트
```
