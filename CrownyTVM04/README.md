## Stage 3 — 셀프호스팅 컴파일러

한선씨 소스코드가 전부 한선씨로 작성된 도구로 컴파일·실행됨:

```
"함수 팩(n) { 만약 (n<2) { 반환 1 } 반환 n*팩(n-1) } 출력(팩(5))"
    ↓ 한선렉서.han (144줄) — 토큰화
    ↓ 한선파서.han (294줄) — AST 생성

    ↓ 한선VM.han (271줄) — 균형3진 VM 실행
    → "120"
```

### Stage 3에서 해결된 핵심 문제

**전역 상수 인라인**: 함수 내부에서 전역 `상수`에 접근 불가 문제 → 컴파일 타임에 상수값을 인라인하여 해결. 이로써 모듈 간 상수 공유가 가능해짐.

**이중 인덱스 대입**: `ctx[0][pc] = val` 구문 미지원 → 임시 변수 패턴으로 대체.

## 부트스트랩 완성도

```
Stage 0 ✅ Rust 점화기 (컴파일러 + VM + 모듈로더 + 상수인라인)
Stage 1 ✅ 한선씨 코어 라이브러리 (10개 모듈, 한선씨로 작성)
Stage 2 ✅ 한선씨로 VM 재구현 (메타순환, 30개 ISA729 옵코드)
Stage 3 ✅ 한선씨 컴파일러를 한선씨로 (렉서+파서+코드젠) ← 완성
Stage 4    CrownyOS 커널 + 셸 (한선씨 전용)
```

### 셀프호스팅의 의미

Stage 3 이후, Rust 코드(3,124줄)는 **점화기(bootstrapper)**로만 존재합니다.
한선씨 코드(1,447줄)가 컴파일러, VM, 표준 라이브러리를 모두 포함하며,
자기 자신의 소스코드를 토큰화→파싱→바이트코드 생성→실행할 수 있습니다.

"외부 언어를 점화용으로만 쓰고, 실제 시스템의 정체성과 확장은 한선씨가 장악한다."
